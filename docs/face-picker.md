# Face Picker & Selected Faces

The face picker lets users select a subset of watch faces, arrange them in a custom order, and view only those faces in a dedicated grid page. The selection persists in the URL across all page navigations.

## Pages

| Page | Purpose |
|------|---------|
| `pick.html` | Face picker — select and reorder faces |
| `selected.html` | Custom grid showing only the user's selected faces |

Both are generated by `build.sh`. `selected.html` is built from `face-template.html` (same as `all.html`), while `pick.html` is a standalone page.

## URL Abbreviations

Each watch face has a 2-letter `urlAbbrev` attribute on its `<watch>` XML element. These abbreviations are used in the `picks` URL parameter to compactly encode the selection.

| Face | Abbrev | Face | Abbrev |
|------|:------:|------|:------:|
| Babylon | `bb` | Basel | `bs` |
| Mauna Kea | `mk` | Firenze | `fi` |
| Haleakalā | `hk` | Venezia | `vz` |
| Hana | `hn` | Terra | `tr` |
| Chandra | `ch` | Miami | `mi` |
| Selene | `sl` | Gaia | `ga` |
| Geneva | `gn` | | |

The `urlAbbrev` is also stored in:
- `Watch` interface in `types.ts`
- `FaceData` interface in `engine-entry.ts`
- Each `face-*.ts` registration call

### Build-Time Uniqueness Check

`build.sh` scans all XML files for `urlAbbrev` attributes and fails the build if any duplicates are found. Any new face must have a unique 2-letter abbreviation.

## The `picks` URL Parameter

The `picks` parameter is a concatenation of 2-letter abbreviation codes with no separator:

```
?picks=bbgntr        → Babylon, Geneva, Terra (in that order)
?picks=mkslfi        → Mauna Kea, Selene, Firenze
```

### Propagation

The `picks` parameter is treated as **opaque cargo** on pages that don't interpret it. All pages preserve it during navigation so the user can always get back to their selection:

- `url-state.ts` — `updateNavigationLinks()` copies the full query string (including `picks`) to all nav links
- `index-page.ts` — `updateLinks()` copies the query string to all face-card links
- Single-face pages, `all.html` — carry `picks` without interpreting it

Only `selected.html`, `pick.html`, and `engine-entry.ts` actually parse the `picks` value.

## Navigation Flow

```
index.html
  ├─ "All Faces" card ────────→ all.html (full grid)
  ├─ "Pick Faces" card ───────→ pick.html (if no picks)
  ├─ "Selected Faces" card ───→ selected.html (if picks exist)
  └─ Individual face card ────→ babylon.html, etc.

pick.html ──── Done ──────────→ selected.html?picks=...
selected.html ─ Edit Picks ──→ pick.html?picks=...
selected.html ─ Click face ──→ babylon.html?picks=...
babylon.html ── 2×2 icon ────→ selected.html (or pick.html if no picks)
babylon.html ── 3×3 icon ────→ all.html
```

### Navigation Icons on Face Pages

The `face-template.html` top bar has three nav icons (left to right):

| Icon | ID | Target |
|------|----|--------|
| 🏠 House | `#back-link` | `index.html` |
| 3×3 dots | `#all-faces-link` | `all.html` |
| 2×2 accent dots | `#selected-faces-link` | `selected.html` or `pick.html` (smart) |

The 2×2 icon uses accent-colored (`#5ca0d3`) dots and is always visible for discoverability. Its destination is set dynamically by `updateNavigationLinks()` in `url-state.ts`.

### Index Page Smart Card

The "Pick Faces" / "Selected Faces" card on `index.html` adapts at runtime:
- **No `picks` in URL** → shows "Pick Faces", links to `pick.html`
- **`picks` present** → shows "Selected Faces", links to `selected.html`

This logic is in `updateLinks()` in `index-page.ts`.

## Pick Page (`pick.html` + `pick-page.ts`)

A standalone page (does not use `face-template.html`) with its own inline CSS matching the app's dark theme.

### Layout

- **Header**: Home link, "PICK YOUR FACES" title, and action buttons (All, None, Reorder, Done)
- **Grid**: All 13 face thumbnails in a responsive flexbox (120px on phone → 180px on desktop)

### Selection UX

- Tapping a face toggles its selection
- Selected faces get a blue border and a numbered badge (1, 2, 3...)
- Unselected faces are dimmed (50% opacity)
- **All** button selects every face in default order
- **None** button clears the selection
- **Done** button navigates to `selected.html?picks=...`

### Reorder Sheet

The **Reorder** button (disabled when < 2 faces selected) opens a slide-down overlay panel:

- Slides in from the top with a 0.3s CSS transition
- Semi-transparent backdrop behind it
- Vertical list of selected faces, each row showing: thumbnail, name, drag handle (≡)
- **Drag to reorder**: Touch (`touchstart`/`touchmove`/`touchend`) and mouse (`mousedown`/`mousemove`/`mouseup`) support
- **Done** button closes the sheet and reorders the picker grid to match

Closing the sheet: tap the backdrop, press Escape, or tap the sheet's Done button.

### Grid Reordering

When the user arrives at the pick page with existing `picks` in the URL, or after using the reorder sheet, the grid is rearranged: **selected faces appear first** (in their current order), followed by unselected faces in the original order. This is done by re-appending existing DOM elements to `#pick-grid` via `reorderGrid()`.

### Face Registry

`pick-page.ts` contains a hardcoded `FACES` array with slug, name, thumbnail path, and abbreviation for each face. This avoids loading/parsing XML just for the picker. The abbreviations must match the XML `urlAbbrev` values.

## Selected Faces Page (`selected.html`)

Built from `face-template.html` with all face scripts (identical to `all.html`). The engine filters and reorders faces at runtime.

### Engine Logic (`engine-entry.ts`)

At startup in `main()`:

1. Detect `selected.html` via `window.location.pathname`
2. If `picks` param is present:
   - Parse the concatenated abbreviation codes
   - Filter `faceDataArray` to matching faces
   - Reorder to match the `picks` order
   - If no faces match, fall back to showing all
3. If no `picks` param: redirect to `pick.html`
4. Add `is-selected-faces` CSS class to `<body>` (instead of `is-all-faces`)

### Edit Picks Button

An "Edit Picks" text link (`#edit-picks-link`) is visible only on `selected.html` via the CSS rule `body.is-selected-faces #edit-picks-link { display: flex }`. It links to `pick.html` with the current query string preserved.

## Adding a New Face

When adding a new watch face:

1. Add a unique 2-letter `urlAbbrev` to the new face's XML `<watch>` element
2. Add `urlAbbrev` to the face's `face-*.ts` registration call
3. Add an entry to the `FACES` array in `pick-page.ts`
4. The build will fail if the abbreviation is not unique
